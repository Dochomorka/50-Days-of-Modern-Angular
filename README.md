# 50 Days of Modern Angular Challenge

Welcome to the **50 Days of Modern Angular** challenge! This repository is an exercise-driven, step-by-step curriculum designed to take you from initial environment setup to architecting enterprise-scale systems. 

Modelled after the popular [30-Days-of-Python](https://github.com/Asabeneh/30-Days-Of-Python) and [30-Days-of-React](https://github.com/Asabeneh/30-Days-Of-React) challenges, this curriculum prioritises the latest industry standards and "Standalone-First" development.

### **🌟 Key Learning Pillars**
*   **Modern Reactivity (Signals):** Master fine-grained reactivity using Writable Signals, Computed values, and Effects to simplify state management and decouple components.
*   **Enterprise Architecture:** Implement **Tomas Trajan’s "Mechanical Boundaries"** to move away from "hope-based architecture." You will learn to enforce a one-way dependency graph using `eslint-plugin-boundaries`.
*   **Professional UI Layer:** Learn to build scalable interfaces by combining the utility-first speed of **Tailwind CSS** with the robust, accessible components of **Angular Material 3**.
*   **Advanced Performance:** Explore the cutting edge of Angular, including **zoneless change detection**, Server-Side Rendering (SSR), and hydration.
*   **Clean Code & Logic:** Deep dive into the `inject()` function, modern functional guards, and state management using the **NgRx SignalStore**.

### **🛠️ How it Works**
Each day in this repository is structured to provide a complete learning loop:
1.  **Core Concepts:** Theoretical deep-dives into modern Angular building blocks.
2.  **Hands-on Implementation:** Step-by-step guides to applying those concepts in a real project.
3.  **Tiered Challenges:** Every day ends with three levels of exercises:
    *   **Easy:** Quick wins to reinforce the day's syntax.
    *   **Medium:** Integration tasks to apply logic to a project.
    *   **Hard:** Architectural puzzles to test your deep understanding.

### **📂 Repository Structure**
The repository is organised by daily folders (`/Days/Day-01`, `/Days/Day-02`, etc.), each containing its own `README.md` guide and solution code.
